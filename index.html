title: nginx +uwsgi 相关配置

tags:

1.环境配置

sudo apt-get install uwsgi-plugin-python (不装的话发起请求的时候uwsgi会报错)
如果是python3 的话请使用sudo apt-get install uwsgi-plugin-python3

2.安装nginx

sudo apt-get install nginx
测试命令：sudo service nginx start

配置nginx服务器

 1：进入/etc/nginx/sites-enabled/;
 2：mv default default.bak （备份文件）
 3：vim default (编辑文件)
 4：server {                                                                       
                listen 80;                   # 服务器监听端口                                                 
                server_name 110.110.110.110; # 域名或者公网IP                                                    
                charset      utf-8;          # 编码                                                  
                client_max_body_size 75M;    # 之前写的关于GET和POST的区别，这里应该是原因之一吧                                                   

                location / {                                                                   
                    include uwsgi_params;         # 导入uwsgi配置                                            
                    uwsgi_pass 127.0.0.1:8000;    # 转发端口，需要和uwsgi配置当中的监听端口一致                                                 
                    uwsgi_param UWSGI_CHDIR /home/chihu; # 项目根目录     
                    uwsgi_param UWSGI_SCRIPT manage:app; # 项目的主程序，即Flask app所在的位置                           
                }                                                                              
            }

输入:wq保存配置文件,可用nginx -t测试一下配置文件是否正确。
若检测配置文件失败，再好好检查下配置文件有没有疏漏。此时访问Nginx服务器会得到502 Bad Gateway的提示，因为请求被Nginx转发了，但是并没有转发服务器来处理请求(还没有配置uwsgi)。

uwsgi安装配置

安装uwsgi

1. sudo apt-get install uwsgi
2. 在项目根目录下新建配置文件 uwsgi.ini（uwsgi支持多种配置文件格式: xml, ini, json等） vim uwsgi.ini
3. [uwsgi]                                                                                                                                                      
    socket = 127.0.0.1:8000             # uwsgi的监听端口                                                                                                                             
    plugins = python                    # 这行一定要加上，不然请求时会出现-- unavailable modifier requested: 0 --错误提示                                                                                                                                  
    chidir = /home/chihu                # 项目根目录                                                                                                                       
    wsgi-file = manage.py               # flask程序的启动文件                                                                                                             
    callable = app                      # 程序变量名    
输入:wq保存配置文件，可以通过uwsgi uwsgi.ini来启动uwsgi。
  • 启动Nginx
    • sudo service nginx restart（每次修改nginx配置问价后，都需要重启nginx服务器来执行新功能）
      uwsgi wsgi.ini

python爬虫基础

发表于 2018-08-03
  • 工作流程
    • 发起请求—>获取响应数据—>解析获取到的数据—>某种格式来保存数据
  • 爬虫定义：
    • 请求网站并提取（满足需求的）数据的自动化程序

  • 爬虫类型：

    • 1.批量型网络爬虫：限制抓取的属性，包括抓取范围、特定目标、限制抓取时间、限制数据量以及限制抓取页面，总之明显的特征就是受限
    • 2.增量型网络爬虫（通用爬虫）：与前者相反，没有固定的限制，无休无止直到抓完所有数据。这种类型一般应用于搜索引擎的网站或程序
      3.垂直网络爬虫（聚焦爬虫）：简单的可以理解为一个无限细化的增量网络爬虫，可以细致的对诸如行业、内容、发布时间、页面大小等很多因素进行筛选。

  • 网络爬虫的搜索策略

    • IP地址搜索策略

      -IP地址搜索策略是先给爬虫一个起始的IP地址。优点：可以搜索的较为全面。缺点：不适合大规模搜索

    • 深度优先搜索策略

    • 宽度优先搜索策略
      -宽度优先搜索策略的优点：一个是保证了对浅层的优先处理，当遇到一个无穷尽的深层分支时，不会导致陷进www中的深层文档中出现出不来的情况发生；另一个是它能在两个HTML文件之间找到最短路径
      -宽度优先搜索的过程是先搜索完一个Web页面中所有的超级链接，然后再继续搜索下一层，直到底层为止。

  • 爬虫能抓怎样的数据？

      1. 网页文本 - 如 HTML、JSON 格式文本等
      1. 图片 - 获取到的是二进制文件，保存为图片格式
      1. 视频 - 同为二进制文件，保存为视频格式即可
      1. 其他 - 只要是能请求道的，都能获取

  • 怎样解析爬取到的数据？

      1. 直接处理 - 比较简单的网页内容
      1. JSON 解析
      1. 正则表达式
      1. BeautifulSoup
      1. PyQuery
      1. XPath

  • 怎样解决 Javascript 渲染的问题？

    • 方法一： 分析 AJAX 请求
    • 方法二： Selenium/WebDriver 来请求
    • 方法三： Splash
    • 方法四： PyV8、Ghost.py

urllib中的urlencode()方法接收的参数为字典类型

  2. python默认的递归次数为1000次，如果递归次数超过则前面递归程序有可能会崩溃

Git

发表于 2018-07-17

文档链接：http://www.cnblogs.com/tugenhua0707/p/4050072.html

一：Git是什么？

Git是目前世界上最先进的分布式版本控制系统。

二：SVN与Git的最主要的区别？

SVN是集中式版本控制系统，版本库是集中放在中央服务器的，
Git是分布式版本控制系统，那么它就没有中央服务器的

三：设置用户名和邮箱作为一个标示

git config --global user.name "";
git config --global user.email ""

四：创建一个版本库（选择你喜欢的地方创建）

git init  ->创建命令
    pwd 命令是用于显示当前的目录。

五：把文件添加到版本库中

git add .(把当前目录下的所有东西都添加上缓存区中)
    使用命令git add ceshi.txt(ceshi为新添的文件) 添加到缓存区中;
    使用git commit提交到github仓库中;
    可以使用 git status 来查看文件
    如果嫌上面显示的信息太多的话，我们可以使用命令 git log –pretty=oneline

六：回退版本

git reset  --hard HEAD^（回退到上一个版本）
如果想回退更高的版本只需要加^ 或者更为简便的git reset  --hard HEAD~100
可以通过如下命令即可获取到版本号：git reflog

七：撤销操作

首先在做撤销之前，我们可以先用 git status 查看下当前的状态。
撤销命令：git checkout  --  readme.txt,
git checkout -- readme.txt 中的 -- 很重要，如果没有 -- 的话，那么命令变成创建分支了。

八：Git与svn创建分支的本质：

git创建分支本质：创建一个文件，写入一个40字节的哈希值
    svn创建分支本质：就是把文件拷贝一份
